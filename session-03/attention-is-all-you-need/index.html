Attention Is All You Need

In 2017, eight researchers asked a radical engineering question: what if a translation model stopped passing information word by word and let every position look directly at the others?

Ashish Vaswani et al. · first posted 2017 · published at NeurIPS 2017. This page explains the architecture and evidence in the paper itself; it is not a history of everything Transformers later became.

1
The 2017 problem

Stop waiting for the previous word

Paper §§1 and 4Table 1

Recurrent networks had to carry information through a chain of hidden states. That made training inherently sequential and gave distant words a long route to one another. The Transformer replaced that chain with self-attention.

One attention calculation

A query meets keys, then carries values

Paper §3.2Equation 1Figure 2

Attention is a weighted lookup. A position makes a query, compares it with keys, converts the scores into weights, and mixes the corresponding values.

Attention(Q, K, V) = softmax(QKT / √dk)V

Why divide by √dk? As vector dimensions grow, unscaled dot products can grow large and push softmax into regions with extremely small gradients. Scaling keeps training better behaved.

The paper's base model used eight attention heads (the big model used sixteen). Each head received its own learned projections of queries, keys, and values, letting the layer form several connection maps in parallel before combining them.

4
The whole Transformer

The title is a provocation, not a parts list

Paper §§3.1–3.5Figure 1

The Transformer removed recurrence and convolution from its encoder–decoder sequence model. It did not remove everything except attention.

Open the architecture layer by layersimplified Figure 1

Encoder · repeated 6×

Input embeddings + positional encoding
↓
Multi-head self-attention
Residual connection + layer norm
Position-wise feed-forward network
Residual connection + layer norm
The encoder lets every input position gather information from the previous layer, then transforms each position through the same feed-forward network. Residual paths preserve the incoming representation; normalization stabilizes the stack.

Decoder · repeated 6×

Output embeddings + positional encoding
↓
Masked multi-head self-attention
Residual connection + layer norm
Attention over encoder output
Residual connection + layer norm
Position-wise feed-forward network
Residual connection + layer norm
Linear layer + output softmax
The causal mask blocks future output positions, so prediction at position i can use only earlier generated outputs. A separate attention block queries the encoder's completed input representations.
Why positional encoding? With neither recurrence nor convolution, order is not built into the architecture. The paper adds sine-and-cosine position signals to embeddings. Why the encoder–decoder bridge? Decoder queries attend to keys and values from the encoder, letting each output position use the full input sequence.

Attention moves information between positions; feed-forward layers transform it; residual connections and normalization make deep stacks trainable; position signals preserve order.

5
What the paper demonstrated

An architecture claim backed by translation results

Paper §4Paper §6Tables 1–2

The paper argued for self-attention using three criteria: computation per layer, how much work can run in parallel, and the path length between positions. It then tested the architecture on translation and parsing.

Table 1, made visiblemaximum path length

For a sequence of length 10, the paper's comparison gives these asymptotic longest paths:

Self-attention
O(1)
Recurrent
O(n)
Convolutional
O(logk n)

Constant path length does not mean free computation: full self-attention compares every pair of positions, giving quadratic sequence-length cost in the paper's formulation.

3.5 days Training time

The big models trained on one machine with eight NVIDIA P100 GPUs.

BLEU is an automatic translation metric based on overlap between a system's translation and reference translations. Higher was better on these 2014 WMT benchmarks, but BLEU is not a general measure of intelligence or even a complete measure of translation quality.

The paper's achievement was specific and substantial: a non-recurrent encoder–decoder trained more efficiently and reached leading results on the tested sequence tasks.

Read the page honestly

Four boundaries worth keeping

Teaching weights ≠ measured weightsThe interactive scores and head lines here are authored diagrams. Paper results and equations are labeled separately.
Attention ≠ a complete explanationA connection map shows information flow and weighting, not every cause of a model's final behavior.
Parallel training ≠ parallel generationTraining can process known sequence positions together; the paper's decoder still generates outputs autoregressively, one position at a time.
The 2017 model ≠ every later LLMThis paper presents an encoder–decoder built for sequence transduction. Many later language models use decoder-only descendants.
