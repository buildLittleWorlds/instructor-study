Where does randomness enter?

Conway's Game of Life can look wild, but with the same starting board and the same rule it always follows the same future. Use this lab to separate three things that are easy to blur together: the starting state, the update rule, and randomness during execution.

The deeper comparison

Two state arrays, three design dials

Shared spine: both systems move through discrete stages by repeatedly transforming an array of states. Life updates a grid of one-bit cells. A transformer updates a sequence of continuous, high-dimensional vectors. That is a real resemblance—but a transformer is not a cellular automaton. The three tools below make the differences inspectable.

1 · How far can one position reach?

Click a cell, then move the step slider. The blue square is the farthest region that cell could have influenced through ordinary local Life updates: a one-cell-per-step “speed of light.” This is a causal bound, not a claim that every shaded cell actually changes.

Illustrative attention reach: one layer

Click a token in the diagram. Standard self-attention permits a token to read any permitted token in the hard context window in one layer. Actual attention weights need not be equal, and a causal mask can forbid looking at future tokens.

Within one transformer layer, the same learned layer function is shared across token positions. Across depth, however, the next layer normally has different learned parameters. Life reuses one rule at every cell and every step.

Interpretation

What matches—and what does not

QuestionGame of LifeLanguage model at inference
Current stateThe living and dead cells on the gridThe prompt and tokens generated so far
What is fixed?The hand-written B3/S23 ruleThe learned model weights that produce next-token scores
Where can randomness enter?The starting board, or an explicitly added noisy updateSampling from the next-token distribution
What does temperature do?Nothing in standard Conway; this lab's noise is not temperatureMakes the sampling distribution sharper or flatter without changing the learned weights

Both systems can produce surprising global structure from repeated state transformations. Both also have a genuine homogeneity: Life applies one rule at every cell, while a transformer applies one shared layer function at every token position within that layer. Neither one CA update nor one transformer layer is universal by itself; universal computation requires iteration over a memory surface that can keep growing—the grid or the autoregressive token tape. Adding intermediate reasoning tokens can therefore give a model more working space without changing its learned weights.

Accuracy guardrail: a transformer is not a cellular automaton. Its neighborhood is global rather than local, its learned rule normally changes from layer to layer, and its internal forward pass has fixed depth rather than an unbounded update loop. Autoregression restores iteration by feeding generated tokens back as external memory.

Method

What is computed, and what is illustrative

Computed in the browser

Life uses ordinary finite-boundary B3/S23. The light cone uses Chebyshev distance: after N local steps, influence can be at most N rows and columns away. The crossing readout uses the selected cell’s farthest board distance. Fingerprints are compact checksums, not scientific measurements.

Illustrative diagrams

The attention lines show architectural permission to connect within a context window, not measured attention weights. The Three Dials diagram is a conceptual design space, not a trace from a trained model. No model, API, account, or collected data is involved.

Limitation

Conway is a counter-model, not a miniature transformer

Life cells hold one bit; transformer positions hold continuous, high-dimensional vectors. Conway’s rule was handwritten and selected by a person; transformer parameters are learned from data. A random starting board is not temperature. The noise slider is an explicitly nonstandard CA variant, while language-model temperature reshapes next-token sampling at inference. It changes neither learned weights nor stored knowledge. These differences are the point of the comparison.
