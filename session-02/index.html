The Garden of Forking Sentences

Dr. Plate — AI + Research, Level 2 · Youth Horizons Learning
Thesis. During sampled generation, a model repeatedly computes next-token scores and a sampler chooses a branch. This toy freezes a small set of possible forks into a tree. Temperature changes how strongly sampling favors the safest branch. Turn it down and the tree approaches a single trunk—the safest sentence nearly every time. Turn it up and more branches become competitive. Difference is easy to buy. Quality is the hard part.
This week’s question: When a system surprises us, did the surprise come from its input, its rule/learned parameters, or randomness while it ran?

Sampling subquestion: what does temperature actually do to the sentences a model can produce—and where does “more surprising” turn into “less coherent”?

Callback: this tree shows sampling paths, not transformer architecture

Interpretation

At a cold temperature the walk usually takes the fattest branch at every fork, so the same safe paths recur. Crank the heat and shy branches (“breathing,” “underwater,” “furious”) win more often—some delightful, some nonsense. The count of distinct sentences climbs, but randomness never invents a new branch. Temperature changes how often this toy visits existing tips; it supplies neither new knowledge nor a quality judgment.

The deeper parallel is iteration over state. Life repeatedly updates a grid; a language model repeatedly extends an external token tape through autoregression. Neither one CA update nor one transformer layer is universal by itself. Universal computation requires iteration over a memory surface that can keep growing—the grid or token tape. Within one transformer layer, the same learned function is shared across positions; across layers, the learned rule normally changes.

Limitation. This is a tiny, hand-built tree with illustrative probabilities; it is not measured from a specific model. A real model recomputes scores over a model-specific vocabulary after each token. The tree omits attention, continuous hidden vectors, learned parameters, and the hard context window. A transformer is not a cellular automaton: it differs in neighborhood, rule across layers, and iteration. Conway’s one-bit handwritten rule does not model learned hidden state; use a neural CA for that comparison.

Try it yourself

In an AI chat, run one prompt five times and count what changes—but do not infer a hidden temperature setting from repetition alone. Then ask for “five versions, from safest to wildest” and judge which differences are merely unusual versus genuinely useful. If your interface exposes temperature, record the setting; otherwise label it unknown.

Temperature changes which branch wins. But before sampling, how did every token help shape those branch weights? If one position can see everything in its context at once, what becomes possible? Next: Attention Spotlight, Session 3.
